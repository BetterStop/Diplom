Ввод пароля.

 

После  запуска  программы на экране появляется окно ввода пароля (рисунок 1).

В поле ввода необходимо ввести пароль и нажать клавишу Enter или кнопку «Подтвердить».

Рисунок 1 – Окно ввода пароля

В случае неправильного ввода система выдаст сообщение об ошибке (рисунок 2) и будет предоставлена возможность повторного ввода.

Рисунок 2 – Сообщение об ошибке

Для выхода из программы необходимо нажать клавишу Esc или кнопку «Отмена». При вводе правильного пароля происходит загрузка главной формы приложения.

 

Главная форма приложения.

 

Главная форма приложения (рисунок 3) представляет собой основную рабочую среду АРМа. В нем выводится информация о сотрудниках, осуществляется управление ими и происходит генерация отчетов.

Главное меню состоит из пяти пунктов:

- файл;

- редактор;

- поиск;

- отчеты;

- справка.

Меню «Файл» (рисунок 4) включает в себя два пункта: «Сменить пароль» и «Выход». Пункт «Сменить пароль» запускает форму смены пароля, при выборе пункта «Выход» происходит выход из программы.

Рисунок 3 – Главная форма приложения

Рисунок 4 – Меню «Файл»

Подменю «Редактор» (рисунок 5) предназначено для работы с карточками сотрудников и со справочниками.

Рисунок 5 – Меню «Редактор»

Пункт «Новая карточка» загружает форму создания новой карточки на сотрудника.

Пункт «Редактирование карточки» загружает для изменений уже существующую карточку сотрудника, предварительно выбранного на главной форме щелчком мыши по полю с данными о сотруднике.

Пункт «Удаление карточки» будет активировано только после выбора сотрудника на главной форме и предназначено для увольнения сотрудника.

Подменю «Поточное редактирование» содержит следующие пункты:

- «Назначение, перемещение» - назначение сотрудника на новую должность, перевод в другой отдел;

- «Семейное положение» - изменение состава семьи сотрудника;

- «Отпуска» -  оформление отпуска сотрудника;

- «Дополнительные данные» - добавление новой дополнительной записи о сотруднике.

Подменю «Редактирование справочников» предназначено для добавления новой, редактирования существующей и удаления ненужной справочной информации. При выборе этого пункта осуществляется запуск формы для работы со справочниками.

Подменю «Поиск» (рисунок 6) предназначено для поиска информации о сотруднике в базе данных.

Рисунок 6 – Меню «Поиск»

Возможен поиск по фамилии, по названию отдела и по должности. При выборе любого пункта поиска на форме появляется поле ввода (рисунок 7) и курсор автоматически перемещается на него. Поиск осуществляется динамически, что позволяет найти похожие поля или всех сотрудников одного отдела и тому подобное.

Рисунок 7 – Поиск по базе данных

Подменю «Отчеты» (рисунок 8) содержит в себе пункты предназначенные для генерации отчетов, а именно:

- «Возрастной состав» - запускает форму анализа возрастного состава предприятия;

- «Национальный состав» - запускает форму анализа национального  состава предприятия;

- «Отчет» - запускает форму для генерации отчета о одном конкретном сотруднике выбранном на главной форме.

Рисунок 8 – Меню «Отчеты»

 

Смена пароля.

 

Форма смены пароля (рисунок 9) предназначена для изменения пароля при входе в программу.

Рисунок 9 – Окно смены пароля

Необходимо ввести старый пароль в поле «Введите пароль», а в поля «Новый пароль» и «Подтверждение» ввести новый пароль. Нажатие на кнопку «Подтвердить» изменяет пароль входа в программу и закрывает форму. В случае неправильного ввода старого пароля система выдаст сообщение об ошибке (рисунок 2.11.10) и будет предоставлена возможность повторного ввода.

Рисунок 10 – Сообщение об ошибке

В случае ввода разных паролей в поля «Новый пароль» и «Подтверждение» система выдаст сообщение об ошибке (рисунок 11) и будет предоставлена возможность повторного ввода.

Рисунок 11 - Сообщение об ошибке

 

Работа с карточкой сотрудника.

 

Форма «Заполнение карточки на нового сотрудника» («Редактирование карточки») (рисунок 12) предназначена для ввода данных о сотруднике в базу данных.

Рисунок 12 – Окно работы с карточкой

Необходимо заполнить все поля данных или выбрать из справочных данных информацию. После заполнения карточки необходимо нажать на кнопку «Ok» и вся информация поместится в базу данных. После этого, если добавлялся новый сотрудник, а не редактировался существующий, программа запросит добавление очередного нового сотрудника (рисунок 13). В случае подтверждения добавления все поля ввода очистятся и можно начинать ввод заново, иначе форма будет закрыта.

Рисунок 13 – Запрос на повторное добавление

Для закрытия формы достаточно будет нажать кнопку «Cancel», но в этом случае введенная информация не будет сохранена.

 

Назначения и перемещения.

 

Окно «Назначения и перемещения» (рисунок 14) предназначено для работы с данными о назначении и перемещении одного выбранного сотрудника. С помощью этой формы сотрудника можно назначить на новую должность или перевести в другой отдел.

Рисунок 14 – Окно «Назначения и перемещения»

 

Оформление отпуска.

 

Окно «Отпуска» (рисунок 15) предназначено для работы с данными об отпусках  одного выбранного сотрудника. С помощью этой формы сотрудника можно оформить отпуск сотрудника.

Рисунок 15 – Окно «Отпуска»

 

Работа с членами семьи.

 

Окно «Члены семьи» (рисунок 16) предназначено для работы с информацией о членах семьи одного выбранного сотрудника.

Рисунок 16 – Окно «Члены семьи»

 

Добавление дополнительной информации.

 

Окно «Дополнительная информация» (рисунок 17) предназначено для работы с дополнительной информацией сотрудника.

Рисунок 17 – Окно «Дополнительная информация»

 

Увольнение сотрудника.

 

Как видно из названия форма «Увольнение сотрудника» (рисунок 18) предназначена для регистрирования увольнения сотрудника.

Рисунок 18 – Форма «Увольнение сотрудника»

В поле «Дата»  вводится текущая дата, а в поле «Основание» причина увольнения сотрудника. При нажатии на кнопку «Уволить»,  происходит перенос всех данных о сотруднике из основной базы в архивную. Делается соответствующая запись в карточке и выводится сообщение об удачной операции.

Для выхода из режима увольнения необходимо нажать на кнопку «Отменить».

 

Редактирование справочников.

 

Форма «Редактирование справочников» (рисунок 20) позволяет редактировать все справочники.

Рисунок 20 – Фома «Редактирование справочников»

Кнопка «Добавить» («Редактировать») вызывает форму добавление (редактирования) записи (рисунок 21). При нажатии на кнопку «Ok» информация заносится в базу. Кнопка «Cancel» закрывает форму без внесения изменений.

Рисунок 21 – Окно добавления записи

         Кнопка «Удалить» - удаляет запись. Система сделает запрос на подтверждение удаления записи (рисунок 22).

Рисунок 22 – Запрос на удаление

         Если запрос на удаление подтвержден – запись удаляется, но если удаляемая запись используется  где-либо в базе, то удаление не возможно. О чем и сообщает (рисунок 23) система и удаление отменяется.

Рисунок 23 – Сообщение о занятой записи

         Кнопка «Применить» - вносит произведенные ранее изменения в базу данных, но не закрывает форму.

         Кнопка «Ok» - вносит произведенные ранее изменения в базу данных, и  закрывает форму.

         Кнопка «Отмена» - Закрывает форму без изменений базы данных.
